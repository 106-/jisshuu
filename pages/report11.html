第十一回レポート

1.initとは?

initとは、linuxを起動した時最初に立ち上がるプログラムである。そのためPIDは1である。

initはシステムを/etc/inittabに書かれているランレベルに変更する。

2.grubとは?

grubとはlinuxがインストールされているコンピュータの電源ボタンを押した時、一番最初に起動するブートローダである。

HDDのMBRにあるgrubが/bootを呼び出し、OSの切り替え画面を表示させる。

設定ファイルは/boot/grub/menu.lstに存在する。

電源ボタンを押すと自動的にGRUBが立ち上がる。そのまま放置していると選択されているOSが起動される。(猶予時間はmenu.lstのtimeoutの値で設定できる。)

"e"を押すと選択されているOSを起動するためのコマンドを変更できる。

"a"を押すとカーネルを起動するときの引数を変えることができる。

"c"を押すとシェルに似たものが表示され、コマンドを実行することができる。(使えるコマンドはTabキーで表示できる。)

ランレベルを変更する場合は、aを押した後に表示されるコマンドの一番最後にランレベルを指定する。

4.upstartの仕組み,/etc/init/*の役割・書式

upstartとはinitと同様にデーモンを起動するデーモンであるが、upstartはイベント(起動した、デバイスが脱着された,Ctrl+Alt+Delが押されたなどの)に対応してデーモンを起動するという点で異なっている。設定ファイルはVineLinuxの場合/etc/init/*.confである。

$ cat /etc/init/control-alt-delete.conf 
# control-alt-delete - emergency keypress handling
#
# This task is run whenever the Control-Alt-Delete key combination is
# pressed.  Usually used to shut down the machine.
start on control-alt-delete
exec /sbin/shutdown -r now "Control-Alt-Delete pressed"

書式は以下の通り

start on どのタイミングで実行するのかの指定。startup(起動時),control-alt-delete,runlevelなどを指定する。
exec コマンドを実行する
respawn 繰り返して起動する
stop on 止めるタイミングを指定する。start onの引数と同じものを指定する。
script,end script シェルのスクリプトを記述できる。

動作の状況は"initctl list"で確認できる。

6.マウントとは?

マウントとは、他のファイルシステムをOSのあるファイルシステムにつなげて、アクセスできるようにすることである。

7.DVDやUSBメモリを入れると自動的にマウントされる仕組み

本来であればautofsの設定をして自動マウントを行う。

しかし、GNOMEが起動している状態ではUSBメモリなどのマウントをGNOMEが行っている。

自動マウントするか否かの設定はgnome-volume-propertiesで行うことが出来る。

8.Linuxで使われているファイルシステム 9.ext2とext3とext4の違い

mkfsできるものを説明する。1EBとは10^6TBである。

名前 最大ファイルサイズ 最大ボリュームサイズ 特徴
ext2 2TB 16TB extを拡張したもの。昔はよく使われていた。ディスクシークが少なくなるように設計されている。fsckが遅い。
ext3 16GB-2TB 2TB-32TB ext2としてもマウントできる。書込している間にfsckできない用になっている。2038年問題に対応していない。
ext4 16TB 1EB 現在の主流である。fsckが早くなった。2514年まで使える。
cramfs 16MB 256MB 読み込み専用のファイルシステムである。インストールイメージに使われている。
msdos 2GB 4GB FAT16とも呼ばれ、MS-DOSで使われていたもの。ファイル名をあまり長くできない。
ntfs 16TB 256TB Windowsで使われているもの。仕様が公開されていないが、有志の働きによりLinuxでも扱えるようになった。
xfs 8EB 8EB 古い歴史を持つ。ファイルの読み書きが早い。削除したファイルは復元できない。小さいファイルが多いと遅くなってしまう。
vfat 2GB 2TB 長いファイル名を使えるようにしたFAT

10.シングルユーザモードとは?

シングルユーザモードとは、余計なデーモンを起動してない状態で(ランレベル1で起動するデーモンのみ起動する)rootとしてシェルを使うモードである。

サーバ不調の解決や、rootパスワードを忘れたときなどに使う。

12.rwでマウントされているディスクにfsckをかける問題

何かのデーモンor誰かがファイルをディスクに書き込んでいる最中にfsckを実行してしまうと、書き込んでいる部分が不正と扱われファイルが破損してしまうため、fsckの実行は書き込み不可の状態で実行すべきである。

13./bin/syncの役割は何か。

Linuxはファイルをディスクに書き込むとき、逐次書き込んでいくのではなくメモリにバッファとして蓄積しておき、後で書き込んでいく。

システムをシャットダウンするときなどには当然このバッファがなく、すべてディスクに書き込まれている状態でなければならない。

そこで/bin/syncコマンドを使う。このコマンドはバッファにあるデータを全てディスクに書き込むコマンドである。

15.中身を移し替えてハードディスクを交換する手順

中身の移し替えにはddコマンドを利用し、直接新しいハードディスクにコピーする。

あるいはディスクイメージを作り、それを復元するという方法もある。

ここでは前者の方法を使う。マウントまでは14番の通り。

// /dev/sdaを/dev/sdb1にコピーする場合。
$ sudo dd if=/dev/sda of=/dev/sdb
