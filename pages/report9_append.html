10./etc/hosts.allowと/etc/hosts.denyの設定

xinetを設定する。まずは、デフォルトでインストールされているProFTPDをxinetdで利用するための設定をする。

/etc/xinetd.d/proftpdを作り、以下を記述する。

service ftp
{
        protocol        = tcp
        socket_type     = stream
        wait            = no
        user            = root
        server          = /usr/sbin/in.proftpd
        disable         = no
}

/etc/proftpd.confを編集する。

(略)
ServerType inetd	//standaloneからinetdに変更する。
(略)
#PASV
MasqueradeAddress	www.elec.e.yamagata-u.ac.jp
PassivePorts    10000   10029
(略)
#PASV
MasqueradeAddress	www.elec.e.yamagata-u.ac.jp
PassivePorts    10000   10029
(略)
(部分を全部コメントアウトする。)
(略)
DefaultRoot	~

ここまででftpサーバの設定は完了した。一旦xinetdを再起動した後、21番ポートを開き手持ちの端末でアクセスすると、無事にフォルダが表示された。

続いて、hosts.allowとhosts.denyを編集する。

2つのファイルを編集し実験した結果を表にまとめる。

以下の表において、接続元は山形大学学内無線LAN(133.24.2.231)である。

hosts.allow hosts.deny 接続可能かどうか
ALL: ALL ALL: ALL 可
in.proftpd: 127.0.0.1 ALL: ALL 不可
in.proftpd: 133.24. ALL: ALL 可
in.proftpd: 133.24. ALL: 133.24. 可

上の結果から、まずallowに書かれているホストは接続を許可され、次にdenyに書かれていないホストの接続も許可され、denyに書かれているホストは接続を許可されていないことがわかる。

実際にftpサーバを運用する場合には、allowに必要なIPアドレスを記述し、denyはALLとしておくのが最適と考えられる。
