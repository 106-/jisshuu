第十回レポート

1.atとは何か。

atとは、コマンドを指定した時刻に実行するプログラムである。

2.atの使用方法を第4回と同様に詳しく説明せよ。

$ at "(時刻)" -f "(コマンド)"

時刻の指定は以下の形式で行う

(時):(分) (日).(月).(年)

-lオプションでジョブの確認ができる。

$ at -l
3       Sat Jun 20 04:00:00 2015 a 106

-dオプションでジョブを削除する。

$ at -d (ジョブ番号)	//番号は-lオプションで見る

3.cronとは何か。

cronとは、コマンドを指定した周期で実行するプログラムである。

4.cronの使用方法を第4回と同様に詳しく説明せよ。

$ crontab -e
(エディタが立ち上がる。エディタに処理内容を記述していく。立ち上がるエディタは変数EDITORの値によって変わる。)

コマンドの記述方法は以下のとおり

(分) (時) (日) (月) (曜日) (コマンド)

"*"は毎を表す。また、"/"を使うと細かく間隔の指定ができるようになる。

曜日の欄には日曜=0として、月曜=1、火曜=2といったように指定する。

(例)

 0 * * * * (コマンド)	//毎時0分にコマンドを実行
 0 0 * * * (コマンド)	//毎日0時にコマンドを実行
 */30 * * * * (コマンド)	//30分ごとにコマンドを実行

5.NTPとは?

NTPとは、コンピュータの時計を他のコンピュータの時計を参照して同期するためのプロトコルである。

階層構造を持ち、上位サーバから正確な時刻を取得することができる。また、通信にかかった時間も考慮して同期している。

UDPの123番ポートを利用している。

6.ntpd、ntpdateの使用方法

ntpd
ntpサーバの役割をするデーモン。

設定ファイルは/etc/ntp.confにある。

以下の様なオプションをntp.confファイルに書き込んでいく。

restrict (ホスト名) mask (サブネットマスク) (フラグなど) 指定したホストとの通信について設定する。defaultを指定したものが基本設定となり、ホストごとに例外を書いていくことになる。
server (ホスト名) minpoll (最小問い合わせ間隔) maxpoll (最大問い合わせ間隔) 指定したホストを参照して時刻合わせをする。minpoll・maxpollのそれぞれには2のべき乗の形で記述する。無くても動く。
fudge ハードウェアクロッククロックから時刻を調整するためのオプション
driftfile 自機と問い合わせ先との差を保存するファイルパスを指定する。
tinker step (秒数1) panic (秒数2) 参考にしているntpサーバとの時刻の差が、秒数1以上であればstepモードで秒数を変更し、秒数2位上であればデーモンを止める。

以下はrestrictに使うフラグである。

ignore 全て接続を無視する。
nomodifty 時刻・設定の変更要求を無視する。
notrust 認証できない場合、通信を無視する。
nopeer 互いに同期をとらない
noquery 時刻の問い合わせを無視する。
notrap trapサービスのパケットを無視する。trapサービスとは、ネットワークを通じて他のコンピュータを監視するためのものである。
ntpdate
ntpサーバに接続し、時刻合わせを行うコマンド。
-q 問い合わせのみ行い、設定はしない
-b stepモード(いきなり正しい時刻に設定するモード)で時刻を設定する。
-B slewモード(ゆっくりと時刻を変更するモード)で時刻を設定する。 
$ sudo ntpdate ntp.elec.e.yamagata-u.ac.jp
ntpq
ntpサーバの現在の設定などを表示するコマンド。

ntp -qで現在同期しているサーバなどの一覧を見ることができる。

$ ntpq -p
     remote           refid      st t when poll reach   delay   offset  jitter


 *ntp-a2.nict.go. .NICT.           1 u   38   64  377   14.550  -28.660  24.932

アスタリスクが付いているサーバが現在参照中のサーバである。

7./sbin/clockの存在理由と使い方

/sbin/clockは/sbin/hwclockにリンクされている。

lrwxrwxrwx 1 root root 7  4月 24 12:41 /sbin/clock -> hwclock*

hwclockはハードウェアクロック(マザーボードが電池で保持している時刻)に触るためのコマンドである。

$ sudo hwclock --show		//ハードウェアクロックの時刻を見る
$ sudo hwclock --systohc	//ハードウェアクロックをシステム時刻に合わせる
$ sudo hwclock --hctosys	//システム時刻をハードウェアクロックに合わせる

8.ntp.elec.e.yamagata-u.ac.jp.を参照する設定にする

restrictで通信を許可し、serverでホストを指定する。

/etc/ntp.confを編集する。

restrict ntp.elec.e.yamagata-u.ac.jp
server ntp.elec.e.yamagata-u.ac.jp

10.閏秒とは?

閏秒とは、原子時計と実際の天体周期との間にある差を埋めるために挿入される一秒のことである。

原子時計は極めて正確に1秒を刻むため、地球の回転とは微妙な差が生じる。その差が0.99秒以下になるように閏秒は挿入されている。

11.前回の閏秒でLinuxに発生した問題

前回の閏秒では、Linuxカーネルで同じ秒数を二回繰り返すことによって閏秒をやり過ごそうとした(59秒を二回数えた)。それを発端としてMySQLやJavaで障害が発生した。

12.閏秒対策

tinkerオプションを0にして、常にslewモードで同期するようにする。

tinker step 0

この設定は、/etc/sysconfig/ntpdに-xコマンドをつけることでも変更できる。

# Drop root to id 'ntp:ntp' by default.  Requires kernel >= 2.2.18.
OPTIONS="-xp /var/run/ntpd.pid"

しかし、このバージョンではプログラムのバグによって、SLEWモードが正しく動かないかもしれないとの情報がインターネットにあった。

( http://hiroki.jp/leap-second-2015 の情報による。)

$ apt-cache policy ntp
ntp:
  インストール済: 4.2.6p3-6vl6
  候補: 4.2.6p3-6vl6
(略)

ntp-4.2.8p3-RC1以降のバージョンであればバグは直っているらしいのだが、おそらくインストールされているntpdにはバグがありそうに思えた。

よって、ここではntp.orgからダウンロードしたソースコードをコンパイルして差し替える。

$ cd ~/ダウンロード
$ wget http://www.eecis.udel.edu/~ntp/ntp_spool/ntp4/ntp-4.2/ntp-4.2.8p3-RC2.tar.gz
$ tar -xvf tar -xvf ntp-4.2.8p3-RC2.tar.gz
$ cd ntp-4.2.8p3-RC2
$ mkdir bin
$ sudo ./configure --prefix=/home/106/ダウンロード/ntp-4.2.8p3-RC2/bin
$ sudo make

しかしここでコンパイルエラーが発生した。

ntp_loopfilter.c:689: error: ‘MOD_NANO’ undeclared (first use in this function)

定数"MOD_NANO"が宣言されていない旨のエラーだ。検索すると、これを"STA_NANO"に置き換えればいいらしい。

置き換えてもう一度makeを実行するとコンパイルに通った。

$ sudo make install
$ cd ./bin/bin
$ sudo service ntpd stop
$ sudo cp ntpd /usr/sbin
$ sudo service ntpd start

/var/log/messagesを見ると、

Jun 26 01:25:28 jisshuu2 ntpd[11857]: ntpd 4.2.8p3-RC2@1.3265-o Thu Jun 25 16:10:03 UTC 2015 (4): Starting

最新版が実行されている。

14.西暦14292年7月6日は何曜日か。求めた方法も記せ。

dateコマンドに、指定した年月日の曜日を求める事ができるオプションがあるのでそれを利用する。

$ date -d 142920706
14292年  7月  6日 水曜日 00:00:00 JST

よって、14292年7月6日は水曜日である。

また、pythonを利用してツェラーの公式で検算した。

( http://www004.upp.so-net.ne.jp/s_honma/doweek.htm を参考にしました。)

Python 2.6.6 (r266:84292, Mar  4 2014, 20:48:09)
[GCC 4.4.5 20101001 (Vine Linux 4.4.5-6.1vl6)] on linux2
Type "help", "copyright", "credits" or "license" for more information.
>>> import math
>>> g = math.floor
>>> h = 142
>>> y = 92
>>> m = 7
>>> d = 6
>>> w = y+g(y/4)+g(h/4)-2*h+g(13*(m+1)/5)+d
>>> w % 7
4.0	//水曜日を表す
