第八回レポート

1.sshとは?

sshとは、暗号を利用して安全に他のコンピュータをリモート操作するためのプロトコルや、それを利用したコマンドのことを言う。

2.コマンド

ssh
sshを利用して他のコンピュータへログインする
$ ssh 106@133.24.255.82
-i 公開鍵認証を利用する。オプションの後に秘密鍵へのパスを指定する。
-p ポート番号の指定
-x/-X X11の転送(リモートのウィンドウをローカルで開く手段)を有効/無効にする。大文字で有効、小文字で無効
-4/-6 IPv4/6の指定
-g ポートフォワード機能を外部から利用できるかの設定 詳しくは10番で
slogin
sshコマンドと同じ。内部でもsshへのリンクとなっている。
lrwxrwxrwx 1 root root 5  6月  5 13:36 /usr/bin/slogin -> ./ssh*
scp
接続先へファイルをコピーする

sshコマンドの欄で挙げたオプションが同じように使える。

-C 通信を圧縮する
-r ディレクトリ内もコピーする時に指定
sftp
sshの上でftpに似たプロトコルでファイルをやり取りするコマンド

ssh-keygen

$ ssh-keygen -t (鍵の種類)

$ ssh-keygen -lf (鍵のパス)

3.共通鍵暗号と公開鍵暗号の比較

共通鍵暗号
暗号化/復号化に使う鍵が同じ暗号。情報の送り手と受け手が遠く離れた場所にいるとき、安全に鍵を渡す手段がない。
公開鍵暗号
暗号化/復号化に使う鍵が違う暗号。方法が複雑で理解しがたいが、不特定多数の相手に対し公開鍵をばらまくことができるため、誰とでも安全な通信ができる。

4.共通鍵を安全に生成する方法

Diffie-Hellman鍵共有という方法で共通鍵を生成・交換する。

最初にpとqという2つの値を決める。pは素数、qはp>qとなる自然数にする。

情報をやり取りする二者はそれぞれでqを何乗かにする。

qをpで割った余りを二者で交換する。

交換した値を再び先ほどと同じ数で何乗かにする。

それをpで割った余りは二者とも同じ値であるので、これを共通鍵とする。

5.公開鍵認証の原理

公開鍵認証には秘密鍵と公開鍵という2つの鍵を利用する。公開鍵は暗号化専用の鍵、秘密鍵は復号化専用の鍵である(RSA鍵の場合を除く)。

まず公開鍵をデータの送り手に送信する。公開鍵は暗号化専用の鍵であるので第三者に盗聴されていても問題ない。

送り手は公開鍵でデータを暗号化し、それを送信する。暗号化されているので当然ながらここで盗聴しても復号できない。

受け手は秘密鍵で復号する。こうして二者の間で安全な通信が実現する。

6.公開鍵認証とパスワード認証の比較

パスワード認証は自分がパスワードを覚えて入ればどこからでもログインすることができる。しかしパスワードが他人に知られてしまった場合、他人がログインすることを許してしまうため危険である。

公開鍵認証は秘密鍵ファイルを指定すればパスワード無しでログインすることができるため、スクリプトの中からの接続がしやすい。しかし逆を言えば鍵が無ければどこからでも入れないため、不便である。

7.RSA鍵、DSA鍵、ECDSA鍵、ED25519鍵の数学的な原理

RSA鍵
素数同士の掛け算は素因数分解しづらいことで実現している鍵方式。公開鍵と秘密鍵が逆にも使える。
DSA鍵
離散対数問題(y=g^x mod pのxをyから求めるのが難しい)から公開鍵認証を実現している。
ECDSA鍵
DSAの強化版。楕円曲線上の離散対数問題(調べたが分からなかった)を応用している。同じ強度の暗号だとしてもDSAよりも短い長さの公開鍵で済む。
ED25519鍵
ツイストしたエドワーズ曲線(Twisted Edwards curve:ax^2+y^2=1+dx^2y^2 という数式で表される曲線らしいが詳しいことは分からなかった)を利用した鍵である。

8.opensshの手動アップデート

$ wget http://updates.vinelinux.org/apt/VineSeed/SRPMS.main/openssh-6.7p1-1vl7.src.rpm
$ sudo rpm --rebuild openssh-6.7p1-1vl7.src.rpm

$ sudo rpm --rebuild openssh-6.7p1-1vl7.src.rpm
$ cd ~/rpm/RPMS/x86_64
$ sudo rpm -Uvh openssh-*.rpm

10.sshのポートフォワード機能とは?

sshのポートフォワードとは、sshサーバの指定したポートに届いた通信を、別のサーバに転送する機能である。

ポートフォワードにはローカル/リモートの2つが存在する。

sshだけでなくhttpやsmtpなどにも使える

  • リモート

リモートコンピュータの特定ポートへの接続を自分に転送する。

(133.24.255.82で)
$ ssh -R 10000:133.24.255.82:22 106@133.24.255.81
106@133.24.255.81's password:
Last login: (略)

(133.24.255.80で)
$ ssh 106@133.24.255.81
106@133.24.255.81's password:
Last login: (略)
$ ssh 106@localhost -p 10000	//ここでのlocalhostは133.24.255.81を指している
106@localhost's password:
Last login: (略)
いらっしゃいませこんにちは
(略)

ちなみに、外部から直接はいることは出来ない。ポートを飛ばした先にログインする必要がある。

$ ssh 106@133.24.255.81 -p 10000
ssh: connect to host 133.24.255.81 port 10000: No route to host
  • ローカル

自分の特定ポートへの接続を他のコンピュータへ転送する。

$ ssh -L 10000:133.24.255.81:80 106@133.24.255.82
106@133.24.255.82's password:
Last login: (略)
いらっしゃいませこんにちは

この状態で、133.24.255.82上で'wget http://localhost:10000/' すると、133.24.255.81のwebページが表示される。


また、sshには-gというオプションが存在し、これを使えば外部からの接続を許すことができる。

(133.24.255.82上でファイアウォールの設定をして、10000ポートを開いた後に)
$ ssh -L 10000:133.24.255.81:80 -g 106@133.24.255.82
(略)

この状態でhttp://133.24.255.82:10000/ を開くと、全てのコンピュータから133.24.255.81のwebページを見ることができる。

11.sshのポートフォワードを通してXを使う

sshコマンドにオプション"-X"をつける

$ ssh -X -i ~/.ssh/authorized_keys 106@133.24.255.81
$ gnome-terminal
(リモートの端末が表示される)
