第十二回レポート

1.IP,TCP,UDP,ICMPとは? TCPとUDPの使い分け

IP
IPとはInternetProtocolの略であり、ネットワークでパケットをやり取りするために使われているプロトコルである。

IPで通信しているコンピュータはIPアドレスを持ち、送られてきたパケットが自分のものであれば受け入れ、そうでなければ破棄する。

TCP
TCPとはTransmissionControlProtocolの略であり、IPの上で信頼性のある通信をするためのプロトコルである。

TCPは送られてきたパケットが正しいものであるのかを確認し、再送を要求する機能を持っているので、情報の欠損のない通信をすることができる。

しかしその信頼性ゆえにDoSなどの脆弱性を受けやすくなっている。

UDP
UDPはUserDatagramProtocolの略である。TCPのような信頼性は無いが、早く情報を伝えることのできるプロトコルである。

UDPにはTCPの持つ、データ欠損を再送要求し補完する機能がないため、信頼性のある通信を行うことはできない。

パケットの内部にはデータが正しいものであるのかを確認するためのチェックサムが存在しているが、そこで検出されたデータ欠損はただ破棄されるだけである。

またTCPにある、送信者と受信者の相互の通信を行わないため、IPアドレスの詐称に弱い。

そのためUDPを利用したプロトコルであるDNSは中間者攻撃に弱くなっている。

また、同一ネットワーク内のホストにブロードキャスト通信(全てのホストに送信)が出来る。

ICMP
ICMPはInternetControlMessageProtocolの略である。接続状況の通知に使われるプロトコルである。

エコー要求/応答の機能や、通信が宛先まで辿り着かなかったことなどの通知を行う機能などを持つ。

ICMPヘッダにはタイプ(そのデータが何を意味するものか)を示す部分、コード(タイプで意味したものが、今どのような状況か)を示す部分が存在する。

エコー通知はpingコマンドで使われている。

  • TCPとUDPの使い分け

TCPはその信頼性を生かしてHTTPなど情報の正確さが重要である通信で使われる。

一方UDPは情報のリアルタイム性が重要である動画配信や音声通信などに利用される。

2.TCPのフラグ,ICMPの種類と役割

  • TCPのフラグ
URG データの中に緊急データが含まれていることを示す。あまり使われていない。
ACK ヘッダに有効なACK番号が含まれていることを示す。ACK番号とはTCPプロトコルでの情報の正確さを確認するための番号である。受信側はシーケンス番号(TCPパケットに付けられる通し番号)に、受信したデータのサイズを足したものをACK番号として送信する。これによって、データがどれだけ正確に届いたかを知ることが出来る。
PSH データをキャッシュせず速やかにアプリケーションに渡すよう要求するためのフラグ
RST TCPでの接続を拒否/中断したいときに使われるフラグ
SYN TCP接続する場合、一番最初に送られるパケットでonになっているフラグ。
FIN 通信が終わったことを示すフラグ。送受信側両方がFINを送信するとTCP接続は終了する。
  • ICMPの種類と役割
タイプ 名前 役割
0 Echo Reply Message(エコー応答通知) 受け取ったエコー応答要求の送信元と送信先を入れ替え、タイプを0にしたものが送信される。pingコマンドで使う。
3 Destination Unreachable Message(宛先到達不可能通知) 送信先のホストに送れない場合の通知。コードにはその理由を示す数字が代入されている。
4 Source Quench Message(送出抑制要求通知) 受信能力を超えた通信であったと通知するもの。
5 Redirect Message(経路変更要求通知) 別のゲートウェイを利用するように要求するもの。受け取ったホストは自身のルーティングテーブルを書き換え、そこに送信するようにする。
8 Echo Message(エコー要求通知) 送信先のノードにエコーを要求する。pingコマンドで使われている。
9 Router Advertisement Message(ルーター広告通知) ルータ(≒デフォルトゲートウェイ)のIPアドレスを通知する。優先度と一緒にルータのIPアドレスが入っている。
10 Router Solicitation Message(ルーター請願通知) ルータにルータ広告を送信するよう要求するもの。
11 Time Exceeded Message(時間切れ通知) IPヘッダのTTL(Time To Live,経由してよいノードの数)が切れても宛先ホストに到達できなかったことを示す。tracerouteコマンドで使われている。
12 Parameter Problem Message(不正引数通知) 問題のあるデータを破棄したことを示す。データにはIPヘッダと問題のあった箇所を示す数字が記入されている。
13 Timestamp Message(タイムスタンプ要求通知) 2つのホスト間の通信にかかる時間を計測するためのもの。0時から経過した秒数をミリ秒込で送信する。
14 Timestamp Reply Message(タイムスタンプ応答通知) ↑の要求に応答するもの。現在時刻を記入して返答する。
15 Information Request Message(情報要求通知) IPヘッダを0(ブロードキャスト通信する意)にして送信する。昔はネットワークに存在するホストの数を計測するために使われていた。今ではネットワークに存在しないIPアドレスをあぶり出す為に使われている。
16 Information Reply Message(情報応答通知) ↑に返答するもの。
17 Address Mask Request Message(アドレスマスク要求通知) アドレスマスク(おそらくサブネットマスクを意味する?)の送信を要求するもの。
18 Address Mask Reply Message(アドレスマスク応答通知) ↑に返答するもの。

3.パケットとは?

パケットとは、情報を細かく分割したものである。情報を細かく分割することによって、データの一部分だけを送りやすくなる(データ欠損があった場合に、該当部分だけを再送しやすい)。また、大きなデータをそのまま送ってしまうと回線がそれだけで埋まってしまい、他のノード同士の通信が不可能になってしまうが、パケットであれば幾つものデータを混在させて通信することができるため、ネットワークでの通信に向いている。

4.パケットフィルタとは?

パケットフィルタとは、送られてきたデータを確認し、指定した条件に一致したパケットのみ通過させ、他のものを破棄するというものである(指定した条件のみ破棄するということもできる。)。

必要のないデータを破棄することで、余計な攻撃を受け付けないようにすることができる。不必要な危険性を回避するために、パケットフィルタは必要だと言える。

5.コマンド使用方法

iptables
パケットフィルタを設定するコマンド。

基本的には

$ sudo iptables (ルールを追加する・削除するなどの指定) (チェイン名) (オプションたくさん)
$ sudo iptables -A INPUT -p tcp -m state --state NEW -m tcp --dport 80 -j ACCEPT

のような形式で実行できる。

-A チェインの最後にルールを追加する。
-D チェインからルールを削除する。
-L チェインのルールを一覧表示する。
-F チェインのルールを全て削除する。
-X チェインを削除する。

チェインとは、いくつかのルールをまとめたものである。まとめることで管理がしやすくなる。

そして、チェイン名の後に指定できるオプションは以下のとおり。

-p プロトコルの指定
-j パケットをどこに送るかの指定。自分で作ったチェインに送るか、ACCEPT(受け入れる)DROP(破棄する)REJECT(拒絶する)LOG(ログに出力)などを指定できる。
-m 使えるオプションを増やすオプション。オプションの後にモジュール名を記入する。詳しくは6番で
iptables-save
iptablesで有効になっているパケットフィルタの状態を標準出力に出力するコマンド。
iptables-restore
パケットフィルタの状態をファイルから読み出すコマンド。

ip6tables,ip6tables-save,ip6tables-restoreは、iptablesの対象がIPv6になるのみでコマンドの使い方はほとんど変っていないため省く。

6.iptablesのモジュールの使用方法

icmp

"--icmp-type"が使えるようになる。icmpのタイプの数値を指定したり、anyで全てのタイプを設定できる。

tcp
--sport,--source-port データの送信元ポートの指定
--dport,--destination-port データの送信先ポートの指定
--tcp-flags TCPヘッダのフラグの指定
udp

tcpと同じ意味を持つ"--source-port"と"--destination-port"が使える

state

接続状態を指定できるモジュール。"--state"オプションが使え、以下のような接続状態が指定できる。

INVALID 既知の状態でない
ESTABLISHED 接続が確立した時
NEW 新しい接続の開始時
RELATED 接続確立後
recent

ものすごい勢いで接続してくるクライアントを指定できる。

--name 名前の指定
--set アドレスをリストに追加する。
--remove リストからアドレスを削除する。
--rcheck リストにIPアドレスが入っているのかのチェックができる。
--seconds アドレスがリストにあり、指定した秒数以内である場合にマッチする。
--hitcount アドレスがリストにあり、指定した回数以上である場合にマッチする。

7.パブリックIPv4アドレスとは? プライベートIPv4アドレスとは?

パブリックIPv4アドレスとは、プライベートIPv4アドレスでなく、また全世界でひとつしか存在しないIPv4アドレスのことである。

プライベートIPv4アドレスとは、LAN内で利用できることになっているIPv4アドレスのことである。

同じLANに属するコンピュータでしか接続することができない。

3つ種類があり、LAN内に存在するコンピュータの数によって使い分ける。

クラスA 10.0.0.0/8
クラスB 172.16.0.0/12
クラスC 192.168.0.0/16

10.ログの記録

( http://wokowa.net/blog/archives/63 )を参考にiptablesの設定ファイルを書く。

 -A RH-Firewall-1-INPUT -d 255.255.255.255 -j DROP	//ブロードキャストアドレスのログ出力を無効にする
 -A RH-Firewall-1-INPUT -d 224.0.0.1 -j DROP			//マルチキャストアドレスのログ出力を無効にする
 -A RH-Firewall-1-INPUT -p udp -m multiport --dport 137,138,139,445 -j DROP
 -A RH-Firewall-1-INPUT -p udp --dport 17500 -j DROP
 -A RH-Firewall-1-INPUT -m limit --limit 1/s -j LOG --log-prefix "[iptables BLOCKED] : " --log-level=info

ip6tablesの設定も同様にする。

 -A RH-Firewall-1-INPUT -d FF01::101 -j DROP
 -A RH-Firewall-1-INPUT -p udp -m multiport --dport 137,138,139,445 -j DROP
 -A RH-Firewall-1-INPUT -p udp --dport 17500 -j DROP
 -A RH-Firewall-1-INPUT -m limit --limit 1/s -j LOG --log-prefix "[ip6tables BLOCKED] : " --log-level=info

これで/var/log/messagesにログが出力されるようになった。
