第五回レポート

1.WWWとは何か。その歴史と意義を述べよ。

WWWとは、主にhtmlで書かれたページ間でリンクを張り、互いにリンクし合ったページ群で出来上がった蜘蛛の巣のようなネットワークのことを指す。

wwwは様々なページの場所の表記をを可能にするURI、ページを構成しそれぞれのページをURIでつなげるHTML、ページの送受信を行うHTTPで成り立っている。

wwwの利点は、html中のリンクを開くことで関連の有るページを開ける点にある。

wwwの歴史は、Tim Berners-Leeが1980年台に個人的なプログラムとして作っていたENQUIREに遡る。

その後、Timは1989年3月にCERNでENQUIREを発展させたシステムの構想の論文を書いた。

1990年11月13日からCERNでwwwの実装が始まった。

1991年8月6日にTimは世界初のwebページを公開した。

(世界で最初に作られたhtmlページを再現したもの: http://info.cern.ch/hypertext/WWW/TheProject.html)

1993年4月30日にwwwページがCERNから開放され、誰にでも利用できるようになった。

2.html、http、httpsとは何か。

html
htmlとはHyperText Markup Languageの略である。

htmlとは、文字を修飾して表示するために作られたテキストファイルである。

htmlファイルはタグとそれを囲む文字列で構成されている。

ユーザはwebサイトにアクセスし、htmlファイルをダウンロードした後、それぞれのブラウザでhtmlファイルを元に画面を作る。

http
httpとはHypertext Transfer Protocolの略である。

主にはファイルの送信に使われている、tcpの上のレイヤに位置するプロトコルである。

例えばgoogleのトップページにアクセスした場合、以下のような通信が行われている。

(送信したもの)
GET / HTTP/1.1
Host: www.google.co.jp
Connection: keep-alive
(略)
(受信したもの)
HTTP/1.1 302 Found
Location: https://www.google.co.jp/?gws_rd=ssl
Cache-Control: private
(略)

それぞれの一行目をステータスライン、それ以下をレスポンス/リクエストヘッダ、ここには無いがレスポンス/リクエストヘッダの下にレスポンス/リクエストボディがある。

受信側のステータスラインにある三桁の数字は、リクエストがどうなったかの状況(成功・ファイルが見つからない・アクセス許可がないなど)を表している。

https
httpsとはHypertext Transfer Protocol Secureの略である。

httpに暗号化などの機能を足したものであり、盗聴されたくないデータのやり取りや、サイトのなりすましをしていない証拠に使われている。

3.httpサーバプログラムには幾つもの種類がある。列挙してそれぞれの特徴を述べよ。

Nginx,IIS,GoAhead WebServerの特徴を述べる。

Nginx

動作が軽量なwebサーバーであり、apacheのようにモジュールを加えることで機能を追加していくことができる。

リバースプロキシやキャッシュなどの機能にも対応している。

IIS

Windows上で動くwebサーバ。設定をGUI上で行うため、コマンドプロンプトに慣れていないユーザでもwebサーバの管理ができる。

http以外にもftpやsmtp等の機能が付いている。

GoAhead WebServer

組込み機器向けのwebサーバ。メモリをあまり使わない。

4.他のhttpサーバと比較してApacheの特徴を述べよ。

Apacheはwebサーバとして利用される機会が高く、設定ファイルの書き方などの日本語ドキュメントが充実しているのが特徴である。

また、ユーザは自由にモジュールを追加することができ、機能を拡張できる。

7.refererとは何か。refererが作られた目的は何か。refererにはどのようなセキュリティ上の問題があるか。

リファラとはhttpのリクエストヘッダにある値の一つである。リファラのおかげでサーバ側はユーザがどのページからリンクされてきたのかがわかるようになっている。

リファラは上記のような統計用途の利用以外にも、画像への直リンクを禁止するなどサイト内のコンテンツを守るためにも利用されている。

しかし、リンク元のURLがわかってしまうというその仕様上、URLのクエリ部分(?以下のもの)がアクセスしたサーバのログに記録されてしまうため危険である。

8.agentとは何か。

agentもまたhttpのリクエストヘッダにある値の一つである。

現在あるブラウザには様々な種類がある。それぞれのブラウザによってhtmlから画面を作る過程は違い、差異が存在する(対応している機能としていない機能など)。その場合、サーバはこのUser-Agentヘッダからユーザがどのブラウザを利用しているかを判断し、適切なページを返している。

どのブラウザのユーザエージェントにもMozilla(NetScapeやFirefoxを開発した団体の名前)が付くのは、さまざまなブラウザが出始めたころにほかのブラウザがNetScapeのUser-Agentを真似していたのを今も戻すことができていない為である。

12.自分で作成したページを、https（443/tcp）でも公開せよ。

mod_ssl,OpenSSLは既にインストールされていた。

本来は信頼されている認証機関から署名された証明書を使うのが望ましいのだが、ここでは自前で発行した証明書を使う。

まずは秘密鍵(*.keyのファイル)を生成する。

(http://vine.1-max.net/apache2-ssl.htmlを参考にした。)

# openssl genrsa -des3 1024 > server.key
Generating RSA private key, 1024 bit long modulus
........++++++
.......++++++
e is 65537 (0x10001)
Enter pass phrase:(パスワードを入力)
Verifying - Enter pass phrase:(もう一度パスワードを入力)

.keyファイルからパスワードを削除する。これをしないと起動時にパスワードが必要になってしまい、起動時に詰まってしまう。

# openssl rsa -in server.key -out server.key
Enter pass phrase for server.key:
writing RSA key

証明書署名要求(*.csrのファイル)を生成する。

# openssl req -new -key server.key -out server.csr
You are about to be asked to enter information that will be incorporated
into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.


Country Name (2 letter code) [AU]:JP(国名)
State or Province Name (full name) [Some-State]:YAMAGATA(都道府県名)
Locality Name (eg, city) []:YAMAGATA(市町村名)
Organization Name (eg, company) [Internet Widgits Pty Ltd]:YAMAGATA Univ.(会社の名前とか)
Organizational Unit Name (eg, section) []:Linux Jisshuu(所属)
Common Name (e.g. server FQDN or YOUR name) []:jisshuu2.elec.e.yamagata-u.ac.jp(サーバのホスト名 ここを間違えるとブラウザに怒られてしまう。)
Email Address []:106@jisshuu2.elec.e.yamagata-u.ac.jp(メールアドレス)
Please enter the following 'extra' attributes
to be sent with your certificate request
A challenge password []:
An optional company name []:　(二行とも空白のままでEnter)

証明書(*.crt)を発行する。

# openssl x509 -in server.csr -out server.crt -req -signkey server.key -days 365
Signature ok
subject=/C=JP/ST=YAMAGATA/L=YAMAGATA/O=YAMAGATA Univ./
OU=Linux Jisshuu/CN=jisshuu2.elec.e.yamagata-u.ac.jp/emailAddress=106@jisshuu2.elec.e.yamagata-u.ac.jp
Getting Private key

実際のWebサーバーでは、秘密鍵と証明書署名要求を自分で作り、証明書署名要求のみを認証局に提出する。認証局は証明書署名要求に署名し、証明書として返す。こうすることで、自分で作成した鍵(秘密鍵・公開鍵の両方)が署名されたことになる。

続いてapache2の設定をする。

# nano /etc/apache2/conf.d/ssl.conf

<VirtualHost _default_:443>
#  General setup for the virtual host
DocumentRoot "/var/www/ssl"
ServerName 133.24.255.82:443
ServerAdmin 106@jisshuu2.elec.e.yamagata-u.ac.jp
ErrorLog logs/ssl_error_log
TransferLog logs/ssl_access_log
 <Directory /var/www/ssl>
  AllowOverride All
  Options ExecCGI
  Order allow,deny
  allow from all
 </Directory>
(略)
SSLCertificateFile /etc/apache2/conf/ssl.crt/server.crt
(略)
SSLCertificateKeyFile /etc/apache2/conf/ssl.key/server.key

apache2を再起動する。

$ sudo apache2ctl restart

tcpの443ポートを開放する。VineLinuxにはiptableの設定をしてくれるファイアウォールが入っているので、その設定を行う。

$ sudo system-config-securitylevel-tui

あとは画面の通りに設定すれば無事にhttpsでの接続ができるようになる。
