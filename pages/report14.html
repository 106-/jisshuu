第十四回レポート

課題をTorに設定した。

1.Torとは?

TorとはThe Onion Routerの略であり、匿名性の高い通信を行う仕組みである。

2.Torが秘匿性の高い通信を行える仕組み

TorネットワークにはOnionRouter・DirectoryServer・DirectoryAuthorityServerが存在する。

情報を送る側はまず最初にDirectoryサーバからOnionRouterの一覧をダウンロードする。

一覧から適当にルータを選び、データを辿らせる適当な順序を決める。

データを経路の逆順に、それぞれのルータに対応した鍵で暗号化する。

データの送り手が送信するときには、データ自体は多重に暗号化された状態にある。この状態がまるで玉ねぎの皮のようであることからOnionRouterと呼ばれている。

送信されたデータを受け取ったルータは、自分の鍵で暗号化された部分を複合し、次のルータへ送る。(玉ねぎの皮を剥いでいく)

それぞれのルータは自分が誰から送られ、誰に送るのかしかわからないため、データの出処を探るには全てのルータに当たる必要がある。

これは現実的ではないため、実質的に秘匿性が高くなっている。

3.Torのビルド・インストール

ここでは、公式サイトからソースをダウンロードしてビルドした。

$ wget https://www.torproject.org/dist/tor-0.2.6.10.tar.gz
$ tar -xzf tor-0.2.6.10.tar.gz
$ cd tor-0.2.6.10
$ sudo ./configure

このままの状態だとライブラリ"libevent"が見つからない旨のエラーが起きるので、apt-getでインストールする必要がある。

checking for libevent directory... configure: WARNING: Could not find a linkable libevent. If you have it installed somewhere unusual, you can > specify an explicit path using --with-libevent-dir

libeventだけでなくlibevent-develも必要らしい。

$ sudo apt-get install libevent-devel

$ sudo ./configure --prefix=/
$ sudo make
$ sudo make install
$ cd ~/tor

4.コマンドの解説

tor
torに接続するためのプログラム
-f 設定ファイルの場所を指定する。
--hash-password コントロールポートに接続するためのパスワードを設定する。
--verify-config 設定ファイルの記述にミスがないか確認する。
tor-resolve
Torを介して名前解決を行うプログラム
-x 逆引きする。
-v 詳細にログを表示する。
-5 SOCKS5プロトコルを利用して通信する。何も指定しないとこれで名前解決しようとする。
-4 SOCKS4aプロトコルを利用して通信する。逆引きができない。
-p ポートを指定する。デフォルトでは9050を使う。 
$ tor-resolve www.elec.e.yamagata-u.ac.jp
133.24.255.82

5.Torを適切に設定する

まずはじめに設定例のファイルをコピーする。

$ sudo cp /etc/tor/torrc.sample /etc/tor/torrc
Socksport (ポート) TorがListenするポートを指定する。アドレス:ポートのようにも指定できる。
SocksPolicy (IP帯) Torに接続してもいいIPアドレス帯を設定できる。上に書かれている設定が優先される。
Log (ログのレベル) (出力先) ログを出力する先を設定する。
RunAsDaemon (1or0) 1の場合、バックで動くような動作になる。
DataDirectory (ディレクトリ) 様々なデータを保存する場所を指定する。デフォルトでは~/.torにある。
ControlPort (ポート) コントロールポートを指定する。
HashedControlPassword (ハッシュ化されたパスワード) コントロールポートに入るためのパスワードを指定する。
HiddenServiceDir (ディレクトリ) HiddenServiceの設定ファイルをおく場所を指定する。
HiddenServicePort (ポート) (IPアドレス:ポート) HiddenServiceを動かすポートを指定する
ORPort (ポート) リレーを受ける場合に使うポートを指定する。NATを超えるために443などがよく使われている?
Nickname (名前) リレーサーバにつける名前
ContactInfo リレーサーバの所有者のメールアドレス
Address リレー時に使うIPアドレスorドメイン名を指定する
DirPort ディレクトリサーバとして使う時に、どのポートをListenするか指定する。
DirPortFrontPage このノードが、TorのディレクトリサーバであるとISPなどに知らせるHTMLファイルのファイルパスを指定する。DirPortで公開される
ExitPolicy Exitノードとして動かすとき、接続できる通信先を限定する。すべてrejectすればExitノードとして動かなくなる。

Logの出力先をsyslogにして、RunAsDaemonの値を1にした。

他の設定は特に変えない。(中継ノードとして働かないように)

6.起動時に自動起動させる

/etc/tor/torrcの"RunAsDaemon"のコメントアウトを外す。

RunAsDaemon 1

今回のTorは自分でビルドしたのでinit.d以下にデーモンの管理をするファイルが存在しない。

そこでapache2のものを参考にして作り、serviceコマンドからの操作できるようにする。

chkconfigにtorを追加し、自動起動させる。

$ sudo chkconfig --add tor
$ sudo chkconfig tor on

7.コントロールポートとは?

コントロールポートとは、torの現在の状態を表示するために開放するポートである。

パスワード認証が使えるが、暗号化はされていないため外部から接続すべきでない。

telnetを利用して接続する。

8.コントロールポートを操作する

torrcのControlPort欄のコメントアウトを外しておく。

パスワードの設定をする。torコマンドの--hash-passwordオプションを利用する。

$ tor --hash-password (パスワード)
Jul 28 14:19:12.092 [notice] Tor v0.2.6.10 (git-58c51dc6087b0936) running on Linux with Libevent 2.0.10-stable, OpenSSL 1.0.1p and Zlib 1.2.5.
Jul 28 14:19:12.092 [notice] Tor can't help you if you use it wrong! Learn how to be safe at https://www.torproject.org/download/download#warning
16:(略)

表示された最後の行をtorrcのHashedControlPasswordの値として貼り付けておき、torを再起動する。

telnetで接続する。

$ telnet localhost 9051

ここまで表示されればコントロールポートを操作できるようになる。

使えるコマンドはたくさんあり、ここで全て解説することはできない。(いくつかのコマンドはここで詳細を見ることができる。http://www51.atwiki.jp/torealy/pages/27.html)

9.HiddenServiceとは?

HiddenServiceとは、torの仕組みを利用して匿名でサーバを立てる仕組みである。

10.実際にHiddenServiceを立ち上げる

torrcを書き換える。

HiddenServerDir /var/tor/hs
HiddenServicePort 80 127.0.0.1:80

$ sudo mkdir /var/tor/hs
$ sudo chmod 0700 /var/tor/hs
$ sudo service tor restart

これで/var/tor/hs/hostnameが生成され、外部からアクセスできるようになった。

$ sudo cat /var/tor/hs/hostname
fl5niq4yczezl4mp.onion

torを介さなくても良い場合は、http://fl5niq4yczezl4mp.onion.to/ から中身を見ることもできる。
